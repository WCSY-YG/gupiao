# 股票选股与买卖点分析项目规划

检索起点：2026-06-10

## 项目目标

建设一个面向 A 股、可扩展到美股/ETF 的量化研究工具，实现：

- 多策略选股：基本面、技术面、资金面、K 线形态、趋势突破、反转、多因子、机器学习评分。
- 买卖点分析：给出入场、加仓、减仓、止损、止盈、失效条件，并解释信号来源。
- 回测验证：验证每个策略的收益、回撤、胜率、风险暴露和稳定性。
- 可视化与报告：展示候选股票、信号解释、K 线指标、回测曲线、风险报告。

本项目只做研究和辅助分析，不输出投资承诺，不默认接入真实交易。

## 总体架构

```text
数据源(AKShare/Tushare/OpenBB)
  -> 数据层(SQLite/DuckDB/Parquet)
  -> 因子与指标层(TA-Lib/ta/pandas)
  -> 策略层(选股规则/多因子/ML)
  -> 信号层(买点/卖点/仓位/风控)
  -> 回测层(backtesting.py/vectorbt)
  -> 报告层(QuantStats/自定义解释)
  -> UI/API(命令行 + Web Dashboard)
```

## MVP 范围

1. 数据：A 股股票列表、日线行情、复权价格、基础财务指标、资金流。
2. 指标：MA、EMA、MACD、KDJ、RSI、BOLL、CCI、ATR、OBV、量价指标、K 线形态、竞价画像。
3. 选股策略：均线多头、放量突破、竞价强度增强、回踩年线、突破平台、低估值高 ROE。
4. 买卖点：技术指标打分、趋势确认、止损止盈、信号失效说明。
5. 回测：单策略回测、批量选股验证、手续费/滑点、T+1/涨跌停约束。
6. 报告：候选股票列表、信号解释、回测指标、最大回撤、收益曲线。

## 策略库规划

- 基本面：PE/PB/ROE、利润增长、现金流、股息率、负债率。
- 趋势类：均线多头、Donchian 突破、相对强度、平台突破、高而窄旗形。
- 反转类：RSI/KDJ/CCI/WR/VR 超买超卖、缩量回踩、ATR 低波动。
- 量价类：放量上涨、资金净流入、涨停后整理、下跌无量。
- 竞价类：高质量小幅高开、竞价放量、竞价价格稳定、竞价强度与近期 K 线共振。
- 形态类：TA-Lib K 线形态、早晨之星、吞没、锤头、乌云盖顶等。
- 多因子：价值、质量、成长、动量、波动率、流动性综合打分。
- ML 类：Qlib 风格因子数据集、LightGBM/MLP/Lasso 预测评分。

## 数据模型草案

- `instruments`：股票基础信息、上市日期、市场、行业、概念。
- `bars_daily`：OHLCV、成交额、换手率、复权因子。
- `fundamentals`：估值、盈利、成长、偿债、股东信息。
- `money_flow`：主力/北向/行业/概念资金流。
- `auction_minutes`：盘前分钟或集合竞价快照、委买委卖、成交量、成交额、最新/参考价。
- `auction_profiles`：竞价缺口、竞价量比、竞价波动、竞价强度评分。
- `indicators_daily`：技术指标和 K 线形态。
- `screening_results`：策略选股结果、评分、命中原因。
- `signals`：买点/卖点/止损/止盈/失效条件。
- `backtest_runs`：回测参数、时间范围、交易成本、结果摘要。

## 技术栈建议

- Python 3.11/3.12。
- 数据处理：pandas，后期可引入 Polars。
- 存储：SQLite/DuckDB + Parquet，先轻量本地化。
- 数据源：AKShare 作为 MVP 主数据源，Tushare/OpenBB/yfinance 作为扩展。
- 指标：TA-Lib 优先，安装受阻时用 bukosabino/ta 兜底。
- 回测：backtesting.py 做 MVP，vectorbt 做批量参数回测增强。
- 报告：QuantStats + 自定义中文解释报告。
- 服务/UI：先 CLI，后 FastAPI + Web Dashboard。

## 风险与约束

- 数据质量：复权、停牌、涨跌停、财务发布日期、指数成分历史变化都要校验。
- 回测偏差：必须防未来函数、幸存者偏差、参数过拟合、偷看财务数据。
- 交易约束：A 股 T+1、涨跌停无法成交、手续费、印花税、滑点要进入假设。
- 策略风险：任何高胜率结果都必须经过样本外验证、滚动验证和压力测试。
- 合规边界：默认只做研究、模拟和报告，不默认真实下单。
